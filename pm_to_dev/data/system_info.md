## 技术栈

- 语言：**Java**（主要）、Python（数据分析/AI）
- 框架：
  - Spring Boot（微服务框架）
  - Spring Cloud（服务治理）
  - MyBatis（ORM框架）
  - Dubbo（RPC框架）
- 数据库：
  - MySQL 8.0（主存储，分库分表）
  - MongoDB 6.0（文档存储）
  - Redis 7.0（缓存/分布式锁）
- 消息队列：
  - RocketMQ（主要消息中间件）
  - Kafka（日志收集）
- 搜索引擎：

### 基础设施
- 服务器：约500台物理机 + 2000个容器实例

- 云服务：
  - 阿里云（主要云服务商）

## 其他信息

### 已有功能模块
- 点歌系统：支持搜索、分类、推荐
- 演唱系统：实时伴奏、人声处理、打分
- 社交系统：关注、动态、K歌房对唱
- 排行榜：日榜、周榜、月榜、地区榜
- 会员系统：VIP权益、订阅管理
- 直播系统：主播直播、互动
- 短视频系统：演唱作品分享、浏览

### 正在开发的功能
- AI智能伴奏调音：根据用户演唱自动调整伴奏
- 实时合唱：多人在线合唱
- 语音AI评分：更精准的演唱评分
- 内容审核AI化：自动识别违规内容

### 技术亮点
- 高并发架构：支持千万级用户同时在线
- 低延迟K歌：端到端延迟<100ms
- 智能推荐：基于用户行为的个性化推荐
- 音频处理：自研音频引擎，音质优秀

### 痛点问题
- 推荐效果：当前推荐点击率约3%，有提升空间
- 新手留存：新用户次日留存偏低，需要优化引导流程
- 内容安全：UGC内容审核压力大
- 成本优化：带宽和存储成本较高，需要优化

### 技术挑战
- 如何在大规模用户下保持系统稳定性
- 如何提升个性化推荐效果
- 如何降低音视频处理成本
- 如何平衡功能开发和代码质量
