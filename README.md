# AI Native 工程师面试题 - 职能沟通翻译助手

这是针对雷石 AI Native 工程师面试题的项目，包含两个互相配合的翻译工具，帮助产品经理和开发工程师更好地互相理解需求。

## 项目概述

### 核心理念

将"翻译"重构为"架构师技术调研工作流"：
- 保留完整的 LangGraph 架构
- 改造提示词以适应不同场景
- 基于实际调研的结果输出

### 两个项目

| 项目 | 功能 | 工作流程 | 知识库 |
|------|------|----------|--------|
| **[pm_to_dev](./pm_to_dev/)** | 产品需求 → 技术方案 | 需求澄清 → 需求简报 → 技术现状调研 → 技术方案报告 | 技术文档 (`data/system_info.md`) |
| **[dev_to_pm](./dev_to_pm/)** | 技术实现 → 产品价值 | 技术澄清 → 技术简报 → 产品价值调研 → 产品价值报告 | 产品知识库 (`data/product_info.md`) |

---

## pm_to_dev - 产品需求到技术方案翻译助手

帮助产品经理和开发工程师互相理解需求，将产品需求转化为结构化的技术方案报告。

### 工作流程

```
产品需求输入
    ↓
需求澄清（Clarify）
    - 澄清业务目标、用户场景、功能范围、时间规划
    ↓
需求简报生成（Research Brief）
    - 整理成结构化需求简报
    ↓
技术现状调研（Research）
    - 并行调研：技术栈、系统架构、数据基础、团队资源
    - 使用 get_system_info 查询内部文档
    ↓
技术方案报告生成（Final Report）
    - 需求理解、技术现状分析、方案对比
    - 架构设计、实施计划、风险评估
    - 报告自动保存到 output/ 目录
```

### 快速开始

```bash
cd pm_to_dev
uv venv
source .venv/bin/activate
uv sync
langgraph dev
```

## dev_to_pm - 技术实现到产品价值翻译助手

帮助开发者和产品经理互相理解，将技术实现转化为结构化的产品价值报告。

### 工作流程

```
技术实现输入
    ↓
技术澄清（Clarify）
    - 澄清对业务和产品的影响（用户场景、功能影响、预期效果）
    ↓
技术简报生成（Tech Brief）
    - 整理成结构化技术简报
    ↓
产品价值调研（Research）
    - 并行调研：公司战略、产品KPI、市场趋势、用户需求、竞品分析
    - 使用 get_product_info 查询产品知识库
    ↓
产品价值报告生成（Final Report）
    - 技术理解、用户价值分析、市场定位
    - 商业模式、推广策略、成功指标
    - 报告自动保存到 output/ 目录
```

### 快速开始

```bash
cd dev_to_pm
uv venv
source .venv/bin/activate
uv sync
langgraph dev
```

## 配置说明

两个项目都需要配置 API 密钥才能使用：

### 环境变量配置

```bash
# OpenAI API（用于 GPT 模型）
OPENAI_API_KEY=your_openai_api_key_here

# Anthropic API（用于 Claude 模型）
ANTHROPIC_API_KEY=your_anthropic_api_key_here
```

### 模型配置（在 Studio UI 中）

| 模型类型 | 推荐模型 |
|----------|----------|
| Research Model | `openai:gpt-4.1` 或 `anthropic:claude-sonnet-4-20250514` |
| Final Report Model | `openai:gpt-4.1` |

## 使用场景

### 场景 1: 产品经理提出新功能

1. 产品经理在 **pm_to_dev** 中输入需求
2. 系统澄清关键信息（业务目标、用户场景等）
3. 系统调研公司技术现状
4. 输出完整技术方案报告

### 场景 2: 开发工程师完成技术实现

1. 开发工程师在 **dev_to_pm** 中描述技术实现
2. 系统澄清产品影响（用户场景、功能影响等）
3. 系统调研市场、用户、竞品信息
4. 输出完整产品价值报告

### 场景 3: 完整工作流（推荐）

1. 产品经理用 **pm_to_dev** 生成技术方案
2. 开发工程师基于方案实现功能
3. 开发工程师用 **dev_to_pm** 提炼产品价值
4. 产品团队基于价值报告制定推广策略
