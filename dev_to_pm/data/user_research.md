# 用户研究知识库

## KTV用户画像

### 基础画像
- **年龄分布**：18-35岁为主（70%）
- **性别比例**：男性55%，女性45%
- **地域分布**：一二线城市用户占比60%
- **消费能力**：月均KTV消费200-500元

### 用户类型
1. **麦霸型**（30%）：热爱唱歌，点歌频繁，对歌曲库要求高
2. **社交型**（40%）：主要为了社交，唱歌是次要的
3. **尝鲜型**（20%）：偶尔尝试，对新功能好奇
4. **预算型**（10%）：对价格敏感，追求性价比

## 用户需求分析

### 核心需求
1. **快速找到想唱的歌**（最重要）
   - 用户平均花2-3分钟找歌
   - 40%用户因为找不到歌而放弃

2. **发现新歌曲**（重要）
   - 用户希望发现符合自己口味的新歌
   - 平均每场KTV尝试3-5首新歌

3. **社交互动**（中等）
   - 希望知道好友在唱什么
   - 希望推荐热门歌曲

4. **个性化体验**（中等）
   - 希望推荐符合自己喜好的歌曲
   - 希望推荐考虑当前场景（时段、氛围）

### 用户痛点
1. **选择困难**
   - 歌曲太多，不知道点什么
   - 推荐不精准，浪费时间

2. **冷启动问题**
   - 新用户不知道自己的喜好
   - 系统不知道用户喜好

3. **推荐不相关**
   - 推荐的歌曲都不喜欢
   - 推荐重复度高

4. **响应慢**
   - 推荐加载时间长
   - 影响用户体验

## 用户使用场景

### 场景一：朋友聚会
- **时长**：2-3小时
- **人数**：4-8人
- **需求**：快速找到大家都能唱的歌
- **痛点**：众口难调，不知道点什么

### 场景二：公司团建
- **时长**：3-4小时
- **人数**：10-20人
- **需求**：活跃气氛，推荐热门歌曲
- **痛点**：需要照顾不同年龄层

### 场景三：情侣约会
- **时长**：1-2小时
- **人数**：2人
- **需求**：浪漫氛围，推荐情歌
- **痛点**：需要选择合适的歌曲

### 场景四：独自练歌
- **时长**：1-2小时
- **人数**：1人
- **需求**：针对性练习，推荐练习曲目
- **痛点**：需要找到适合练习的歌曲

## 用户行为数据

### 点歌行为
- 平均每场点歌：15-20首
- 点歌来源分布：
  - 搜索：60%
  - 推荐：20%
  - 热门榜：15%
  - 其他：5%

### 用户偏好
- **语言偏好**：中文70%，英文20%，其他10%
- **年代偏好**：近3年歌曲占50%
- **类型偏好**：流行60%，摇滚20%，民谣15%，其他5%

### 使用习惯
- 高峰时段：周五晚、周六晚、周日晚
- 平均使用时长：2.5小时
- 重复使用率：月均2-3次

## 用户反馈

### 正面反馈
1. 推荐准确度高（相关性>70%）
2. 响应速度快（<100ms）
3. 推荐有惊喜感

### 负面反馈
1. 推荐重复度高
2. 冷启动不友好
3. 推荐不考虑场景
4. 无法理解推荐理由

### 改进建议
1. 增加推荐多样性
2. 提供推荐理由
3. 支持手动调整推荐偏好
4. 考虑当前时段和场景
